## 🚀 Philosophy

This documentation follows the **YAGNI principle** - only essential documentation that serves a clear purpose:
- **Current and accurate** - reflects the actual project state
- **Actionable** - provides clear guidance for development decisions
- **Minimal** - avoids redundant or outdated information
- **Living** - evolves with the project

## 📝 Contributing

When adding documentation:
1. **Is it essential?** - Does it solve a real problem?
2. **Is it accurate?** - Does it reflect the current state?
3. **Is it maintainable?** - Will it stay current over time?
4. **Is there a better place?** - Could this be code comments or inline documentation?
