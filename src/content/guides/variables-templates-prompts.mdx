# Variables et Templates dans les Prompts Pharmaceutiques

## Objectifs d'apprentissage

À la fin de ce guide, vous serez capable de :

- Comprendre comment utiliser des variables pour personnaliser vos prompts
- Créer des prompts réutilisables avec des informations qui changent
- Maîtriser la syntaxe des variables selon votre outil d'IA
- Organiser vos prompts de manière efficace et professionnelle

## Prérequis

- Avoir lu le guide "Structuration de Prompts avec XML pour la Pharmacie"
- Comprendre les concepts de base de l'ingénierie de prompts
- Avoir une expérience pratique avec au moins un modèle d'IA

## Concepts clés

### Qu'est-ce qu'une variable ?

Une variable est un "placeholder" dans votre prompt qui sera remplacé par une valeur spécifique à chaque utilisation. C'est comme un formulaire que vous remplissez avec les informations du patient.

**Exemple simple :**

```
{{NOM_MEDICAMENT}}=Minoxidil
{{AGE_PATIENT}}=45
{{POIDS_PATIENT}}=70

---

Analyse les interactions médicamenteuses pour {{NOM_MEDICAMENT}} chez un patient de {{AGE_PATIENT}} ans pesant {{POIDS_PATIENT}} kg.
```

### Pourquoi utiliser des variables ?

Les variables vous permettent de :

- **Réutiliser** le même prompt pour différents patients
- **Personnaliser** vos analyses selon le contexte
- **Standardiser** vos approches tout en gardant la flexibilité
- **Accélérer** la création de nouveaux prompts

## Comment utiliser les variables dans vos prompts

### Étape 1 : Définir vos variables au début

Commencez toujours par définir toutes vos variables au début du prompt :

```
{{NOM_MEDICAMENT}}=Atorvastatine
{{DOSE}}=20mg
{{AGE_PATIENT}}=67
{{POIDS_PATIENT}}=75
{{FONCTION_RENALE}}=Clairance créatinine 35ml/min
{{ALLERGIES}}=Aucune connue
{{INDICATION}}=Hypercholestérolémie

---
```

### Étape 2 : Utiliser les variables dans votre prompt

Ensuite, utilisez ces variables dans votre demande :

```
Analyse la posologie de {{NOM_MEDICAMENT}} {{DOSE}} pour un patient de {{AGE_PATIENT}} ans pesant {{POIDS_PATIENT}} kg.

Considère que :
- La fonction rénale est : {{FONCTION_RENALE}}
- Les allergies sont : {{ALLERGIES}}
- L'indication est : {{INDICATION}}

Propose des ajustements si nécessaire et recommande un monitoring adapté.
```

## Bonnes pratiques

### 1. Organisation des variables

- **Groupez par catégorie** : Données patient, médicaments, contexte
- **Utilisez des noms clairs** : `AGE_PATIENT` plutôt que `age`
- **Maintenez la cohérence** : Même format partout
- **Séparez clairement** : Utilisez `---` pour séparer variables et instructions

### 2. Nommage des variables

- **Utilisez des noms explicites** : `{{MEDICAMENT_DOSE}}` plutôt que `{{dose}}`
- **Évitez les abréviations** : `{{FONCTION_RENALE}}` plutôt que `{{FR}}`
- **Utilisez des underscores** : `{{SURFACE_CORPORELLE}}` plutôt que `{{SURFACECORPORELLE}}`

### 3. Documentation des variables

- **Décrivez clairement** le rôle de chaque variable
- **Spécifiez le format** : mg, kg, années, etc.
- **Indiquez si c'est requis** ou optionnel
- **Fournissez des exemples** de valeurs valides

## Pièges à éviter

### 1. Variables mal nommées

**À éviter :**

```
{{a}}=45
{{m}}=Atorvastatine
{{d}}=20mg
```

**Préférer :**

```
{{AGE_PATIENT}}=45
{{NOM_MEDICAMENT}}=Atorvastatine
{{DOSE}}=20mg
```

### 2. Oubli de définir les variables

**À éviter :**

```
Analyse les interactions pour {{NOM_MEDICAMENT}} chez un patient de {{AGE_PATIENT}} ans.
```

**Préférer :**

```
{{NOM_MEDICAMENT}}=Warfarine
{{AGE_PATIENT}}=67

---

Analyse les interactions pour {{NOM_MEDICAMENT}} chez un patient de {{AGE_PATIENT}} ans.
```

### 3. Variables trop complexes

**À éviter :**

```
{{PATIENT_DONNEES_COMPLETES_AVEC_ANTECEDENTS_ET_TRAITEMENTS}}=[Tout le dossier patient]
```

**Préférer :**

```
{{AGE_PATIENT}}=67
{{ANTECEDENTS}}=HTA, diabète
{{TRAITEMENTS}}=Lisinopril, metformine
```

Les variables transforment vos prompts en outils puissants et réutilisables. En maîtrisant cette technique simple, vous pourrez :

1. **Accélérer votre travail** en réutilisant des structures éprouvées
2. **Standardiser vos approches** tout en gardant la flexibilité
3. **Réduire les erreurs** grâce à une organisation claire
4. **Collaborer efficacement** en partageant des templates validés

Commencez par créer des prompts simples avec quelques variables, puis progressez vers des structures plus complexes. N'oubliez pas que la clé du succès réside dans la clarté et l'organisation de vos variables.

**Rappel important** : Adaptez toujours la syntaxe de vos variables à l'outil d'IA que vous utilisez pour optimiser les performances.
