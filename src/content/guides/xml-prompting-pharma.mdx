# Structuration de Prompts avec XML pour la Pharmacie

## Objectifs d'apprentissage

À la fin de ce guide, vous serez capable de :

- Comprendre pourquoi utiliser les balises XML pour structurer vos prompts
- Maîtriser la syntaxe des balises XML sémantiques
- Créer des prompts clairs et précis pour les tâches pharmaceutiques
- Organiser vos prompts de manière hiérarchique et logique

## Prérequis

- Avoir lu le guide "Prompting : Les Bases"
- Comprendre les concepts fondamentaux de l'ingénierie de prompts
- Avoir accès à un modèle Claude 4 ou équivalent

## Concepts clés

### Pourquoi utiliser XML ?

L'utilisation de balises XML est devenue la meilleure pratique en 2025, particulièrement recommandée par Anthropic pour les modèles Claude 4. Cette approche transforme radicalement la façon dont les prompts sont structurés et interprétés.

**Avantages clés :**

- **Clarté** : Séparation nette des différentes sections du prompt
- **Précision** : Réduction des erreurs d'interprétation par le modèle
- **Flexibilité** : Modification facile des sections sans réécriture totale
- **Parseabilité** : Facilité d'extraction de parties spécifiques des réponses

### Balises essentielles

Les balises essentielles à maîtriser sont :

- `contexte` : pour les informations de base
- `instructions` : pour ce que vous voulez que le modèle fasse
- `contraintes` : pour les limites et exigences
- `format_attendu` : pour la structure de réponse souhaitée

## Méthodologie

### Étape 1 : Identifier la structure de votre prompt

Avant de commencer, décomposez votre demande en sections logiques :

- **Contexte** : Informations de base nécessaires
- **Instructions** : Ce que vous voulez que le modèle fasse
- **Contraintes** : Limites et exigences spécifiques
- **Format attendu** : Structure de la réponse souhaitée

### Étape 2 : Créer des balises sémantiques

Utilisez des noms de balises qui ont du sens dans votre domaine :

**Exemples de bonnes balises :**

- `contexte_pharmaceutique`
- `instructions_analyse`
- `exemples_cliniques`

**Exemples à éviter :**

- `text1`
- `data`
- `info`

### Étape 3 : Organiser hiérarchiquement

Structurez vos balises de manière logique et imbriquée. Voici un exemple de structure :

```xml
<analyse_medicament>
  <nom_molecule>{{NOM_MOLECULE}}</nom_molecule>
  <propriétés>
    <pharmacocinétique>
      <biodisponibilité>{{VALEUR}}</biodisponibilité>
    </pharmacocinétique>
  </propriétés>
  <interactions>
    <interaction type="medicament">
      <substance>{{SUBSTANCE}}</substance>
      <niveau_risque>{{NIVEAU}}</niveau_risque>
    </interaction>
  </interactions>
</analyse_medicament>
```

## Exemples pratiques

### Exemple 1 : Demande d'information pharmaceutique

```xml
<task>
  Analyser les interactions médicamenteuses potentielles
</task>

<contexte>
  <patient>
    <age>67</age>
    <sexe>M</sexe>
    <antécédents>HTA, Diabète type 2</antécédents>
  </patient>

  <médicaments_actuels>
    <médicament>
      <nom>Lisinopril</nom>
      <dose>20mg/jour</dose>
    </médicament>
    <médicament>
      <nom>Metformine</nom>
      <dose>850mg 2x/jour</dose>
    </médicament>
    <médicament>
      <nom>Amlodipine</nom>
      <dose>5mg/jour</dose>
    </médicament>
  </médicaments>

  <nouveau_médicament>
    <nom>Spironolactone</nom>
    <dose>25mg/jour</dose>
    <indication>Insuffisance cardiaque</indication>
  </nouveau_médicament>
</contexte>

<instructions>
  1. Identifier les interactions pharmacocinétiques potentielles
  2. Évaluer les risques d'hyperkaliémie
  3. Proposer des recommandations de monitoring
  4. Suggérer des alternatives si nécessaire
</instructions>

<format_sortie>
  <analyse_complète>
    <interactions_détectées>
      <interaction>
        <médicaments_concernés>[Liste]</médicaments_concernés>
        <mécanisme>[Description]</mécanisme>
        <niveau_risque>[Élevé/Moyen/Faible]</niveau_risque>
      </interaction>
    </interactions_détectées>
    <recommandations>
      <monitoring>[Liste des paramètres à surveiller]</monitoring>
      <alternatives>[Suggestions d'alternatives]</alternatives>
    </recommandations>
  </analyse_complète>
</format_sortie>
```

## Bonnes pratiques

### 1. Nommage sémantique des balises

- Utilisez des noms qui reflètent le contenu réel
- Évitez les abréviations obscures
- Maintenez une cohérence dans tout votre projet

### 2. Imbrication hiérarchique

- Organisez vos balises de manière logique
- Évitez les niveaux d'imbrication trop profonds (max 4-5 niveaux)
- Utilisez des balises intermédiaires pour regrouper des informations

### 3. Consistance dans l'utilisation

- Utilisez les mêmes noms de balises dans tout votre projet
- Référencez les balises dans vos instructions
- Documentez votre structure de balises

### 4. Gestion des variables

- Utilisez une syntaxe cohérente pour les variables
- Documentez clairement chaque variable
- Fournissez des exemples de valeurs

## Pièges à éviter

### 1. Balises trop génériques

**À éviter :**

- `info`
- `data`
- `text`

**Préférer :**

- `contexte_patient`
- `données_cliniques`
- `instructions_spécifiques`

### 2. Imbrication excessive

**À éviter - trop de niveaux :**

```xml
<analyse>
  <patient>
    <données>
      <personnelles>
        <identité>
          <nom>Dupont</nom>
        </identité>
      </personnelles>
    </données>
  </patient>
</analyse>
```

**Préférer - structure plus plate :**

```xml
<analyse>
  <patient>
    <nom>Dupont</nom>
    <age>45</age>
    <poids>70kg</poids>
  </patient>
</analyse>
```

### 3. Oubli de la documentation

- Documentez toujours vos balises
- Expliquez le rôle de chaque section
- Fournissez des exemples d'utilisation

### Exercice 1 : Créer un prompt pour l'analyse de posologie

Créez un prompt XML pour analyser la posologie d'un médicament chez un patient pédiatrique.

```xml

  Analyser la posologie pédiatrique d'un médicament
</task>

  <patient>
    <age>8</age>
    <poids>25kg</poids>
    <surface_corporelle>0.95m²</surface_corporelle>
  </patient>

  <médicament>
    <nom>Paracétamol</nom>
    <forme>Sirop</forme>
    <concentration>24mg/ml</concentration>
  </médicament>

  <indication>Fièvre 39.2°C</indication>

<instructions>
  1. Calculer la posologie recommandée selon le poids

  4. Donner des conseils d'administration
</instructions>

<format_sortie>
  <posologie_calculée>
    <dose_par_prise>[mg]</dose_par_prise>
```
